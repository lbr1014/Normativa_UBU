\apendice{Documentación de usuario}

\section{Introducción}

En este manual se incluye una guía destinada a que el usuario final conozca el funcionamiento de la \eng{web} PythIA. Concretamente se van a describir los requisitos que debe cumplir el usuario para utilizar la aplicación (ver sección~\ref{usu:req}) así como el método de instalación de la aplicación para el usuario final (ver sección~\ref{usu:inst}). Finalmente, se incluye un manual explicando todas las funcionalidades de la aplicación (ver sección~\ref{usu:manual}).

\section{Requisitos de usuarios}
\label{usu:req}

Para utilizar la aplicación PythIA, el usuario únicamente necesita disponer de acceso a Internet y de un navegador \eng{web} moderno compatible con los estándares actuales~\cite{usuario:webStandards}. Se recomienda utilizar un navegador con visor \file{PDF} integrado para visualizar o descargar los documentos \file{PDF} de las licitaciones, 

La aplicación ha sido probada satisfactoriamente en los navegadores \file{Google Chrome}, \file{Mozilla Firefox} y \file{Microsoft Edge}. Concretamente se han probado en las siguientes versiones de los navegadores:
\begin{itemize}
	\item \file{Google Chrome} 137.0 a  148.0.
	\item \file{Mozilla Firefox} 139.0 a 150.0.
	\item \file{Microsoft Edge} 137.0 a 148.0.
\end{itemize}

Para acceder a las funcionalidades del sistema es necesario tener una cuenta de usuario registrada y autenticarse mediante correo electrónico y contraseña.

Respecto al \eng{hardware} no existen requisitos específicos, puesto que PythIA ha sido desarrollada siguiendo un diseño \eng{responsive}. Esto permite su utilización desde diferentes tipos de dispositivos, incluyendo ordenadores de sobremesa, ordenadores portátiles, tabletas y teléfonos móviles, adaptando automáticamente la interfaz al tamaño de pantalla disponible. 

\section{Instalación}
\label{usu:inst}

No es necesario instalar ningún \eng{software} adicional, ya que la aplicación se ejecuta a través del navegador \eng{web}. Solo se tiene que acceder a la dirección \url{https://pythia.es/} e iniciar sesión con sus credenciales. En caso de no disponer de una cuenta, deberá registrarse.

Las actualizaciones de la aplicación se realizan de forma centralizada en el servidor, por lo que los usuarios siempre acceden a la versión más reciente sin necesidad de efectuar ninguna acción adicional en sus dispositivos.

Si el usuario introduce credenciales incorrectas, la aplicación mostrará un mensaje de error indicando que el correo electrónico o la contraseña no son válidos, permitiendo el reintento.

En la parte superior de la página se muestran diferentes indicadores y estadísticas relacionadas con el funcionamiento del sistema. Entre ellos se incluyen las métricas globales de evaluación del sistema \eng{RAG}, obtenidas mediante herramientas como \eng{RAGAS}, que permiten conocer aspectos como la fidelidad de las respuestas, la relevancia de la información recuperada o la cobertura del contexto utilizado. Pulsando el botón <<Ver detalles>> se puede acceder a la pantalla de \textbf{Evaluar modelos} (ver sección~\ref{manual:evaluar}). También se presentan estadísticas de uso de los distintos \eng{LLM} (\eng{Long Large Models}) disponibles en la aplicación, así como un selector que permite elegir el modelo con el que se desea realizar las consultas. Pulsando el botón <<Comparar los modelos>> que se muestra debajo del selector se puede acceder a la página de \textbf{Comparar modelos} (ver sección~\ref{manual:compararModelos}).

La página dispone de dos modos diferentes de consulta, \texttt{Chat} (ver figura~\ref{fig:Manual_usuario/Consulta_chat}) y \texttt{Formulario}(ver figura~\ref{fig:Manual_usuario/Consulta_formulario}). El \texttt{Chat} permite realizar consultas en lenguaje natural de forma libre. Para ello, el usuario únicamente debe introducir su pregunta en el cuadro de texto y pulsar el botón <<Preguntar>>. El sistema procesará la consulta, recuperará los fragmentos documentales más relevantes y generará una respuesta utilizando el modelo de lenguaje seleccionado. Por su parte el \texttt{Formulario} está orientado a usuarios que desean realizar consultas guiadas sin necesidad de redactar preguntas manualmente. En primer lugar, se debe seleccionar el expediente sobre el que desea realizar la consulta mediante la lista desplegable disponible. Se puede cribar entre documentos administrativos o técnicos desde el desplegable inferior. También existe la posibilidad de realizar una consulta global sobre todos los expedientes seleccionando la opción correspondiente. A continuación, el usuario puede indicar el tipo de consulta que desea realizar. Es posible activar la opción <<Solicitar resumen del documento>>, mediante la cual el sistema genera automáticamente un resumen completo del expediente seleccionado sin necesidad de seleccionar una pregunta específica (ver figura~\ref{fig:formulario_opciones}). Una vez configurados los parámetros deseados, el usuario debe pulsar el botón <<Preguntar>>.

\subsection{Comparar modelos}
\label{manual:compararModelos}

La página de comparativa de modelos permite analizar el comportamiento de los distintos modelos de lenguaje (\eng{LLM}) disponibles en PythIA a partir de estadísticas reales de uso obtenidas durante el funcionamiento de la aplicación (ver figura~\ref{fig:Manual_usuario/Comparar_modelos}).

\imagendiagram{Manual_usuario/Comparar_modelos}{Pantalla con gráficas para comparar los modelos \eng{LLM} disponibles en la aplicación \eng{web} PythIA.}{1}

En la parte superior se muestran diferentes indicadores globales, entre los que se incluyen el número total de modelos disponibles, el número de consultas realizadas, la cantidad total de \eng{tokens} procesados y el tiempo medio de respuesta registrado. Estos indicadores ofrecen una visión general del rendimiento global del sistema.

A continuación, la página presenta diversas gráficas comparativas que permiten analizar el comportamiento de cada modelo desde diferentes perspectivas. Entre ellas se incluyen el número de veces que ha sido utilizado cada modelo, el volumen de \eng{tokens} procesados, los tiempos medios de respuesta y el dispositivo usado por \file{Ollama} para responder (\file{GPU/CPU}). Esta información sirven para identificar qué modelos ofrecen mejores prestaciones en términos de velocidad, coste computacional o frecuencia de uso.

Pulsando el botón <<Volver a consultar>>, situado en la parte superior derecha de la página, el usuario puede regresar directamente a la \textbf{Página de consultas} (ver sección~\ref{manual:pagConsulta}).


\subsection{Evaluar modelos}
\label{manual:evaluar}

Los resultados más relevantes de la última evaluación se muestran en la página de consultas mediante gráficas de anillos que resumen las métricas de manera visual. Concretamente representan la fidelidad, la relevancia de la respuesta, la corrección de la respuesta, la precisión del contexto, la cobertura del contexto y la relevancia del contexto. Solo los administradores pueden ejecutar una nueva evaluación. Para ello disponen del botón <<Evaluar RAG>>, que inicia la ejecución completa del \eng{pipeline} de evaluación (ver ~\ref{fig:evaluacion}). Durante el proceso se muestra una barra de progreso que informa del porcentaje completado y del estado actual de la evaluación (ver figura~\ref{fig:Manual_usuario/administrador/Evaluacion_rag2}).
